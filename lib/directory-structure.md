# lib 目录结构说明

本文档用于说明 `lib` 目录下的结构与职责边界，便于团队协作与后续维护。

## 目录树（规划）

```
lib
├─ api            # 存放请求
├─ assets         # 存放资源
├─ components     # 存放公共组件
├─ contants       # 存放常量文件
├─ viewmodels     # 存放类型文件
├─ pages          # 存放页面
├─ routes         # 存放路由配置
├─ stores         # 存放全局状态组件
├─ utils          # 存放工具类
└─ main.dart      # 入口
```

## 目录职责说明

- api：封装网络请求、接口定义与调用入口，统一错误处理与数据解析。
- assets：静态资源（图片、图标、字体等）聚合管理，按模块归类。
- components：可复用的基础与复合组件，保持无业务或弱业务耦合。
- contants：项目常量、枚举值等集中管理，避免散落定义。
- viewmodels：视图模型与类型定义，承载页面/组件所需的数据结构。
- pages：页面级组件，组织业务流程与交互逻辑。
- routes：路由表与导航相关配置，集中声明路由路径与守卫策略。
- stores：全局状态与模块化子状态（如主题、用户、购物车等）。
- utils：工具函数与通用库（格式化、校验、适配层等）。
- main.dart：应用入口，初始化配置与根组件挂载。

> 备注：若当前仓库尚未创建上述子目录，可按需逐步补齐；目录名以团队约定为准。
